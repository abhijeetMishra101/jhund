# Contributing to Jhund

Thanks for taking the time to contribute! Here's how to get a change into the codebase.

## Before You Start

- Check the open issues to see if someone is already working on the same thing.
- For anything larger than a small bug fix, open an issue first so we can discuss the approach.

## How to Submit a Pull Request

1. **Fork the repo** — click the Fork button at the top of this page to get your own copy.
2. **Create a branch** — use a short, descriptive name:
   ```
   git checkout -b fix/my-bug-fix
   ```
3. **Make your changes** — keep commits small and focused. One logical change per commit.
4. **Write a clear commit message** — describe *what* changed and *why*, not *how*.
5. **Push your branch**:
   ```
   git push origin fix/my-bug-fix
   ```
6. **Open a pull request** — go to the repo on GitHub, click **Compare & pull request**, and fill in the template.

## Pull Request Checklist

- [ ] The change does one thing and does it well.
- [ ] Existing functionality is not broken.
- [ ] The pull request description explains the problem and the solution.
- [ ] Any new behaviour is covered by tests (if applicable).

## Code Style

- Follow the conventions already in the codebase — consistency matters more than personal preference.
- If you're unsure, just ask in the pull request and we'll work it out together.

## Questions?

Open an issue and tag it with the `question` label. We're happy to help.
