## Context

Three scheduled jobs were specified in the original Phase 7 and never built:

1. **Standup** — daily 9am: Riley collects updates from all active bots
2. **Retrospective** — end-of-sprint: Riley synthesises cross-team patterns
3. **Message archiving** — weekly: moves messages >90 days old to Storage (DB bloat prevention)

Archiving is a reliability concern — without it the `messages` table grows unbounded (~24M rows/year at 1,000 workspaces).

All three use Vercel Cron (defined in `vercel.json`).

## 1. Standup Cron

### Route: `GET /api/cron/standup`

**Schedule**: `0 9 * * 1-5` (9am UTC, weekdays)

**Logic**:
1. Fetch all workspaces
2. For each workspace:
   - Find the `#standup` channel (name = 'standup') — skip if not found
   - Find Riley (role_key = 'ops') for that workspace
   - Insert a system message: `"📋 Good morning — time for your daily standup. What's everyone working on today?"`
   - For each active bot role (non-ops, has a non-archived channel):
     - Call `respondToMessage(channelId, workspaceId)` in that bot's channel with context: "It's standup time. Summarise what you're working on today in 2–3 sentences."
     - Collect each bot's response and post a summary in #standup authored by Riley
   - Update `workspaces.last_standup_at = NOW()`

**Simplified flow (Phase 11 — no cross-bot orchestration yet)**:
- Riley posts the opening message in #standup
- Each bot responds independently in its own channel (not collected into standup — that's workflow chains, Phase 12)
- Riley posts a "Standup kicked off — check each channel for updates" message

This is the MVP standup. Workflow chains (Phase 12) upgrade it to a collected summary.

### Security
- Route must verify `Authorization: Bearer ${CRON_SECRET}` header
- Vercel sets this automatically for cron routes; add `CRON_SECRET` env var

## 2. Retrospective Cron

### Route: `GET /api/cron/retrospective`

**Schedule**: `0 17 * * 5` (5pm UTC, Fridays)

**Logic**:
1. Fetch all workspaces
2. For each workspace:
   - Find the `#retrospective` channel — skip if not found
   - Find Riley for that workspace
   - Fetch last 7 days of messages across all channels (limit 200, ordered by created_at desc)
   - Build a summary prompt: "You are Riley. Based on the following team activity from the past week, write a brief retrospective: what went well, what was challenging, what's coming up. Keep it under 200 words and use plain English."
   - Call Claude with that prompt (not `respondToMessage` — direct Anthropic call, no channel context)
   - Post Riley's retrospective summary in #retrospective
   - Update `workspaces.last_retro_at = NOW()`

## 3. Message Archiving Cron

### Route: `GET /api/cron/archive-messages`

**Schedule**: `0 2 * * 0` (2am UTC, Sundays)

**Logic**:
```
For each workspace:
  For each channel:
    SELECT * FROM messages
    WHERE channel_id = $channelId
    AND created_at < NOW() - INTERVAL '90 days'
    ORDER BY created_at ASC
    LIMIT 5000

    If rows found:
      Compress to JSON (gzip)
      Upload to Storage: archives/{workspaceId}/{channelId}/{YYYY-MM}.json.gz
      DELETE FROM messages WHERE id IN (archived_ids)
```

**Key constraints**:
- Delete only after successful Storage upload (never delete before confirming write)
- Batch deletes in chunks of 500 to avoid long transactions
- Log counts: `[archive] workspace={id} channel={id} archived={n} rows`
- Never archive messages referenced by a pending `plan_id` (join check before delete)

### Storage bucket
Name: `message-archives` | Access: private | No public URLs

### Module: `lib/crons/archive.ts`

```typescript
export async function archiveOldMessages(): Promise<{ archived: number; workspaces: number }>
```

## Cron Auth Pattern (all three routes)

```typescript
export async function GET(request: Request) {
  const auth = request.headers.get('authorization')
  if (auth !== `Bearer ${process.env.CRON_SECRET}`) {
    return NextResponse.json({ error: 'Unauthorized' }, { status: 401 })
  }
  // ... cron logic
}
```

Add `CRON_SECRET` to Vercel env vars — generate with `openssl rand -base64 32`.

- Returns correct `{ archived, workspaces }` counts
